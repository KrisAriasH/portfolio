Carta I — Full Stack

ATTIMO

Plataforma de organización académica — full-stack

Laravel React MySQL API REST Sanctum Auth Figma

ATTIMO es una plataforma responsiva para que estudiantes y profesores organicen cursos, actividades y evaluaciones desde un solo lugar.

Arquitectura

API REST en Laravel 10 consumida por un frontend en React, con capas separadas de modelos, controladores y rutas.

Base de datos

~17 tablas en MySQL: usuarios, cursos, carreras, grupos y actividades, más tablas pivote para relaciones muchos-a-muchos (carreras-cursos, usuarios-grupos, actividades-usuarios/grupos/carreras).

Backend

Autenticación con Laravel Auth y Sanctum para la API. Endpoints REST para actividades, grupos, categorías y usuarios (registro, login, recuperación de contraseña), más un módulo de encuestas con estadísticas agregadas por consulta directa a la base de datos.

Frontend

Dashboard en React con calendario de actividades, múltiples cursos por usuario, modo oscuro y panel de estadísticas con gráficos.

Reto técnico

Modelar correctamente las relaciones muchos-a-muchos entre actividades, grupos, carreras y usuarios sin duplicar lógica, y exponer esas relaciones como estadísticas agregadas de forma eficiente.

Roles & alcance

Vistas diferenciadas para estudiante (progreso, próximos eventos) y profesor (reportes de carga de actividades por grupo).

Carta II

SINABI — Sistema Nacional de Bibliotecas

Rediseño y reestructuración de sitio institucional

UX Research Arquitectura de información UI Redesign Figma

El sitio del Sistema Nacional de Bibliotecas de Costa Rica (Ministerio de Cultura y Juventud) tenía menús sobrecargados de opciones, lo que complicaba encontrar servicios básicos como buscar en el catálogo o ubicar una biblioteca cercana.

Propuse una nueva estructura de navegación, agrupando y reduciendo las categorías del menú principal, y rediseñé la página de inicio para darle más peso a las acciones que la mayoría de usuarios necesita primero. Respeté al máximo la identidad visual existente y ajusté la paleta de colores para reforzar la jerarquía.

Antes
Después

Antes

  • Menú principal con demasiadas categorías visibles a la vez
  • Jerarquía poco clara entre servicios y contenido informativo
  • Acciones clave (buscar, ubicar biblioteca) diluidas en la página
  • Arquitectura de información basada en la organización interna

Después

  • Estructura de menú simplificada y reagrupada por intención de uso
  • Accesos directos a las 4 acciones más buscadas desde el inicio
  • Paleta ajustada para mejorar contraste y lectura

Carta IV

Inkpixell

Blog experimental responsivo con estética retro

UI Design Responsivo Modo oscuro Figma HTML/CSS

Inkpixell es un blog con estética Y2K, inspirado en las páginas personales de principios de los 2000. Diseñé todo el sistema visual en Figma —tipografía pixelada, íconos hechos a mano, paneles tipo "widget"— antes de programarlo, cuidando que la nostalgia visual no sacrificara la usabilidad.

El sitio es completamente responsivo e incluye modo claro y oscuro, ambos manteniendo la misma identidad visual retro.
